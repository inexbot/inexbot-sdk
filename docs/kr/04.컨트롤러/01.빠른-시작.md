## 1.2 编译套件설치

以下编译链根据具体的컨트롤러硬件选择설치：

### 1.2.1 gcc-4.8 编译套件설치

因为 Ubuntu 20.04 自带的源 gcc 最低버전无法설치4.8。我们필요添加 trusty 的源。

```bash
sudo sh -c 'echo "deb http://dk.archive.ubuntu.com/ubuntu/ trusty main" >> /etc/a pt/sources.list'
sudo sh -c 'echo "deb http://dk.archive.ubuntu.com/ubuntu/ trusty universe" >> /e tc/apt/sources.list'
```

```bash
sudo apt update
sudo apt install -y gcc-4.8 g++-4.8
sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-4.8 40
sudo update-alternatives --install /usr/bin/g++ g++ /usr/bin/g++-4.8 40
sudo apt-get install -y gcc-4.8-multilib g++-4.8-multilib
```

다운로드 lib32 库파일

解压lib32.xxx파일，并替换原有的/usr/lib32파일夹。

```bash
tar -zxvf lib32.xxx
sudo rm -rf /usr/lib32
sudo mv usr/lib32 /usr/
```

### 1.2.2 gcc-9.4 编译套件설치

Ubuntu20.04 的源直接설치即可。

```bash
sudo apt install gcc g++
```

### 1.2.3 aarch64-linux-gnu-gcc-5.3 编译套件설치

다운로드编译套件：链接: gcc-linaro-5.3.1-2016.05-x86_64_aarch64-linux-gnu.tar.gz 提取码: w33y

新建T5⽂件夹，将gcc-linaro-5.3.1-2016.05-x86_64_aarch64-linux-gnu.tar.gz解压进去。

```bash
mkdir ~/T5
tar -zxvf gcc-linaro-5.3.1-2016.05-x86_64_aarch64-linux-gnu.tar.gz -C ~/T5
```

添加环境변수

```bash
gedit ~/.bashrc
```

将下面这行拷贝到最后，保存退出即可。

```bash
export PATH=$PATH:$HOME/T5/gcc-linaro-5.3.1-2016.05-x86_64_aarch64-linux-gnu/bin
```

# 2. 构建项目

## 2.1 项目初始化

可以사용VS code或者其它IDE来개발。

新建一个 demo 파일夹，将头파일放到include파일夹下，将 libNexRob.a 放到lib파일夹下。，目录的结构참조如下：

```text
├── include
│   ├── nrcAPI.h
│   └── nrcAPI_advance.h
├── lib
│   └── libNexRob.a
└── src
    └── main.cpp
```

## 2.2 main.cpp파일

```cpp
#include <iostream>
#include "nrcAPI.h"


void SystemStartup() {
  //输出Nexmotion버전库信息
  std::cout << "库버전：" << NRC_GetNexMotionLibVersion() << std::endl;
  //启动제어시스템
  NRC_StartController();
  //检测제어시스템是否初始化完成
  while (NRC_GetControlInitComplete() != 1 ) {
    NRC_Delayms(100);   //延时100ms
  }
  //清除所有错误
  NRC_ClearAllError();


  std::cout << "----" << NRC_GetControlInitComplete() << std::endl;
  std::cout << "StartController Success" << std::endl;
  std::cout << "获取同步버전号" << NRC_GetSyncVersion() << std::endl;
  NRC_Delayms(200);
}


int main() {
  //시스템启动
  SystemStartup();

  std::cout << "Hello World" << std::endl;


  //保持프로그램继续실행
  while(true) {
    NRC_Delayms(2000);
  }
}
```

## 2.3 编译代码

### 2.3.1 直接编译

确认自己的컨트롤러시스템架构，选择对应的编译链도구，下面사용的是gcc-4.8，根据架构不同，可以换成gcc-9。

> 注：如果是x64架构的컨트롤러필요将g++-4.8修改为g++-9，-m32修改为-m64，如果컨트롤러是x86架构的可不用进行修改。

```bash
g++-4.8 -m32 -o nrc2.out src/*.cpp -I./include -L./lib -lNexRob -lpthread -lm -ldl -lrt -std=c++11
```

### 2.3.2 사용Makefile

新建Makeifile파일

在demo目录下新建Makefile파일，将下面内容拷贝到Makefile파일中，并保存。

```makefile
TARGET=nrc2.out
all :
	g++-4.8 -m32 -o nrc2.out src/*.cpp -I./include -L./lib -lNexRob -lpthread -lm -ldl -lrt -std=c++11


clean :
	rm $(TARGET) $(objects)
```

Makefile파일建好之后在编译时只필요在demo파일目录下打开终端输入make即可编译컨트롤러可执行프로그램

在demo目录下打开一个新的终端，输入make，编译完成后在demo目录下有一个nrc2.out的可执行프로그램

# 3. 远程登陆到컨트롤러

## 3.1 确认电脑연결上컨트롤러

首先确保你的电脑已经跟컨트롤러연결了통신，电脑的IP필요跟컨트롤러的IP在同一网段下。컨트롤러的出厂默认IP为192.168.1.13，所以电脑연결컨트롤러的网口同样필요설정为1网段（比如可以설정192.168.1.110）。설정完成之后在Ubuntu中打开一个新的终端输入

```bash
ping 192.168.1.13
```

出现下面如图所示，即为同컨트롤러통신통신成功：

![Ubuntu 终端中执行 ping 192.168.1.13 成功收到컨트롤러应答的통신测试截图](assets/ks-02.png)

如出现不成功的情况，请检查网线的연결，电脑的IP，컨트롤러的IP。

# 3.2 사용ssh登陆到컨트롤러

```bash
ssh inexbot@192.168.1.13
```

然后密码输入123，即可进行远程제어了。

# 3.3 查看컨트롤러시스템架构

登陆进컨트롤러后，可以通过uname -a查看컨트롤러的架构

![SSH 远程登录到컨트롤러后执行 uname -a 명령查看컨트롤러시스템架构的终端输出截图](assets/ks-03.png)

图中所示是三个i686，分别为机器硬件名，处理器유형，硬件平台。i686我们俗称为32位시스템。

# 4. 将프로그램上传到컨트롤러실행

## 4.1 사용SCP上传프로그램

编译完成后，我们将要把프로그램给替换进컨트롤러当中실행我们自己编译的2차 개발프로그램，继续在当前终端下输入

```bash
scp nrc2.out inexbot@192.168.1.13:~/robot
```

输入完成后会出现一句 “ inexbot@192.168.1.13's password: “。

我们继续敲123即可，当出现下图就표시프로그램已经프로그램替换进컨트롤러当中了。

![사용 scp 명령将 nrc2.out 上传到컨트롤러的终端输出截图，显示上传进度条和完成信息](assets/ks-04.png)

当然也会出现替换不成功的时候，终端会显示Text file busy，这种情况就是컨트롤러原本的nrc2.out프로그램还在실행，无法被替换。

这就필요ssh远程登陆到컨트롤러的后台中将正在开启的nrc2.out进程给关闭。

```bash
sudo killall -9 nrc2.out
```

进程关闭后就可以再次执行上面提到的scp的조작，就可以成功把2차 개발프로그램替换到컨트롤러当中。

## 4.2 远程登陆到컨트롤러中启动프로그램

随后就可以执行我们编译的2차 개발프로그램，，在컨트롤러后台的robot目录下输入

```bash
sudo ./nrc2.out
```

执行的效果如下如：

![在컨트롤러后台 robot 目录下执行 sudo ./nrc2.out 启动2차 개발프로그램的终端실행效果截图](assets/ks-05.png)

到这一步我们的2차 개발项目已经成功실행起来了。
