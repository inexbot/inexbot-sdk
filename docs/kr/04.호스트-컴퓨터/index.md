# 호스트 컴퓨터

libnrc_host는 Socket_API 통신 프로토콜을 기반으로 캡슐화된 C++ 라이브러리로, 호스트 컴퓨터에서 컨트롤러 기능을 빠르게 통합할 수 있는 네트워크 인터페이스를 제공하며, Linux/Windows 멀티 플랫폼 및 다양한 고급 언어를 지원합니다.

## 기술 사양

| 항목 | 설명 |
|------|------|
| 시스템 지원 | Windows, Linux, x86, x86_64, arm, arm64 |
| 포트 설명 | 6000: 제어/조회; 6001: 티치 프로그래밍; 7000: 서보 추적 데이터 |
| API 문서 | https://doc.hmilib.inexbot.coision.cn/nrc__interface_8h.html |

## 문서 구조

```
호스트 컴퓨터
├── Python/
│   ├── 시작하기/         Python 프로젝트 처음부터 구성하기
│   └── API 예제/
│       ├── 기본 응용/       연결, 단일 포인트 모션, 연속 궤적, 도구 캘리브레이션, 드래그 티치
│       └── 고급 응용/       servo_move 추적, 메시지 콜백
├── C++/
│   ├── 시작하기/         MinGW+Qt / MSVC / Linux 환경 초기화
│   ├── API 예제/
│   │   ├── 기본 응용/       좌표계 위치, 단일/연속 궤적, 티치/드래그, 도구 캘리브레이션, 곡선 모션
│   │   └── 고급 응용/       servoJ 관절 제어, servo_move 추적, 서보 포인트 위치 제어
│   └── API 참조.md         API 온라인 문서 링크
└── C#/
    ├── 시작하기/         Windows Forms 애플리케이션 초기화
    └── API 예제/            연결 예제
```

## 언어 비교

| 언어 | 적용 시나리오 | 컴파일러/프레임워크 | 특징 |
|------|----------|-------------|------|
| Python | 빠른 검증, 스크립트 | Python 3.x (SDK 버전과 일치 필요) | 글루 언어, 통합 테스트에 적합 |
| C++ | 고성능, 실시간 제어 | MSVC 2017+, MinGW64, gcc aarch64 | 최고 성능, 제품급 통합에 적합 |
| C# | Windows 데스크톱 앱 | .NET Framework 4.8, .NET 8 | WinForms/WPF, 데스크톱 통합 용이 |
