# 주석

## 변수

변수/바인딩 변수의 JSON 구조:

```json
{
    "data": 3.0,
    "secondvalue": 0,
    "value": 0,
    "varname": ""
}
```

| 매개변수 | 유형 | 설명 |
|------|------|------|
| data | float | 숫자형 데이터 |
| secondvalue | int | 바인딩 변수 여부, 0=아니오, 1=예 (형식: GD[GI0002]) |
| value | int | 데이터 유형, 0=숫자형 (data 노드 입력), 7=문자열 (varname 노드 입력), 8=변수 또는 바인딩 변수 (varname 노드 입력) |
| varname | string | 문자열 유형 또는 변수명 |

| 매개변수 | 유형 | 설명 |
|------|------|------|
| ctype | int | 포인트 유형, 1=P, 2=E, 3=S, 4=R, 5=GP, 6=GE |
| key | string | 포인트 이름 |
| note | string | 포인트 주석 |
| data | array | 21비트 배열, 상세는 아래 설명 참조 |
| paraVarData | array | 21비트 매개변수 변수 배열, 구조는 변수 노드와 동일 |

**data 배열 설명 (21비트):**

| 비트 | 설명 |
|----|------|
| 1번째 | 좌표계: 0=관절, 1=직교, 2=도구, 3=사용자 |
| 2번째 | 단위: 0=도 단위, 1=라디안 |
| 3번째 | 좌우수(형태): 0=임의, 1=좌, 2=우 |
| 4번째 | 도구 번호 |
| 5번째 | 사용자 번호 |
| 6~7번째 | 예비 |
| 8~21번째 | 좌표 값 (7비트 내부 축 + 7비트 외부 축) |

> 참고: 존재하지 않는 노드는 기본값 0 또는 ""
