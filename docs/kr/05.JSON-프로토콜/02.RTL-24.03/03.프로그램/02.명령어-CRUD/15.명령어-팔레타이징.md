# 명령어-팔레타이징제어류

## 1. 팔레타이징시작 (PALON)

**설명**：팔레타이징 시작의 판단, 변수 제어 코드를 수정하여 몇 번째 층, 몇 번째 작업물인지 판단할 수 있음.

### 매개변수 설명

| 매개변수 이름 | 유형 | 설명 |
|--------|------|------|
| ID | int | 팔레타이징 프로세스 번호, 값 범위 [1,99] |
| STACKTYPE | int | 팔레타이징 유형, 값 범위 [0,1]<br/>0：팔레타이징<br/>1：언팔레타이징 |
| _pCurrentLayersNum | 변수 객체 | 현재 층 팔레타이징 수；int 유형 변수 또는 바인딩 변수, 전역 또는 지역<br/>해당 변수의 값이 string 유형 [-]를 나타낼 때：사용하지 않음 |
| _pLayersNum | 변수 객체 | 현재 팔레타이징 층；int 유형 변수 또는 바인딩 변수, 전역 또는 지역<br/>해당 변수의 값이 string 유형 [-]를 나타낼 때：사용하지 않음 |
| _pWorkpieceSum | 변수 객체 | 현재 적재된 작업물 총수；int 유형 변수 또는 바인딩 변수, 전역 또는 지역<br/>해당 변수의 값이 string 유형 [-]를 나타낼 때：사용하지 않음 |
| _stackSequence | int | 단층 팔레타이징/언팔레타이징 순서, 값 범위 [0,1]<br/>0：정방향<br/>1：역방향 |
| logout | bool | 주석 처리 여부<br/>true：작업 파일에서 해당 명령어가 실행되지 않음<br/>false：주석 처리되지 않음, 실행 가능 |
| type | int | PALON은 열거 열 type에서 70 |
| userParamInt | int | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 2차 개발에서 사용자 정의, 현재 의미 없음 |

### 예시

```json
{
  "ID": 1,
  "STACKTYPE": 0,
  "_pCurrentLayersNum": {"data": 0.0, "secondvalue": 1, "value": 4, "varname": "GI[GI001]"},
  "_pLayersNum": {"data": 0.0, "secondvalue": 1, "value": 1, "varname": "I[I001]"},
  "_pWorkpieceSum": {"data": 0.0, "secondvalue": 0, "value": 1, "varname": "[-]"},
  "_stackSequence": 1,
  "logout": false,
  "type": 70,
  "userParamInt": 0,
  "userParamString": ""
}
```

## 3. 팔레타이징 진입점 (PLAENTER)

**설명**：그리퍼 선택, 팔레타이징 진입점.

### 매개변수 설명

| 매개변수 이름 | 유형 | 설명 |
|--------|------|------|
| ID | int | 팔레타이징 프로세스 번호, 값 범위 [1,99] |
| ParaACC | 변수 객체 | 가속도；int 유형 또는 double 유형 변수 또는 바인딩 변수, 전역 또는 지역；값 범위 [1,100] |
| ParaDEC | 변수 객체 | 감속도；int 유형 또는 double 유형 변수 또는 바인딩 변수, 전역 또는 지역；값 범위 [1,100] |
| ParaLINE | 변수 객체 | 게이트형 변위 거리；직접 입력 또는 int 유형 또는 double 유형 변수 또는 바인딩 변수, 전역 또는 지역；값 범위 [-5000,5000]<br/>보간 방식으로 게이트형을 선택할 때 사용 |
| ParaPL | 변수 객체 | 평활；int 유형 변수 또는 바인딩 변수, 전역 또는 지역；값 범위 [0,5] |
| ParaTIME | 변수 객체 | 사전 실행 시간；int 유형 또는 double 유형 변수 또는 바인딩 변수, 전역 또는 지역 |
| ParaSTRAIGHT | 변수 객체 | 게이트형 직선 거리；직접 입력 또는 int 유형 또는 double 유형 변수 또는 바인딩 변수, 전역 또는 지역；값 범위 [0,5000] |
| ParaV | 변수 객체 | 모션 속도；int 유형 또는 double 유형 변수 또는 바인딩 변수, 전역 또는 지역<br/>모션 방식이 관절 모션일 때：m_vUnit 값은 2, 속도 단위는 %；값 범위 [1,100]<br/>모션 방식이 원호 또는 직선일 때：m_vUnit 값은 1, 속도 단위는 mm/s；값 범위 [1,1000] |
| Posture | int | 자세, 값 범위 [0,2]<br/>0：비활성<br/>1：수동 쓰기<br/>자동 계산 자세 |
| logout | bool | 주석 처리 여부<br/>true：작업 파일에서 해당 명령어가 실행되지 않음<br/>false：주석 처리되지 않음, 실행 가능 |
| moveType | string | 모션 보간 방식<br/>MovJ：관절<br/>MovL：직선<br/>MovC：원호<br/>MovArch：게이트형 |
| para_posture_ | 변수 객체 | 자세 수동 쓰기 값；int 유형 또는 double 유형 변수 또는 바인딩 변수, 전역 또는 지역 |
| type | int | PLAENTER는 열거 열 type에서 73 |
| offsetAxis | int | 변위축, 값 범위 [0,2]<br/>0：X축<br/>1：Y축<br/>2：Z축 |
| userParamInt | int | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| xyOpt | int | XY축 모션 경로 최적화, 값 범위 [0,1]<br/>0：비활성<br/>1：활성 |
| zOpt | int | Z축 모션 경로 최적화, 값 범위 [0,1]<br/>0：비활성<br/>1：활성<br/>Z축 모션 경로를 최적화하려면 팔레타이징 전에 고정점 하나를 삽입해야 함 |

## 4. 팔레타이징 보조점 (PALSHIFT)

**설명**：팔레타이징 보조점.

**높이 규칙 설명**：
- 진입점 높이가 고정점보다 낮을 때, 진입점의 높이는 고정점과 보조점과 동일한 값 선상에 놓임 (측면에서 보면 동일 직선, 평면에서 보면 동일 직선이 아님, XY축 불변)
- 진입점 높이가 고정점과 보조점 사이에 있을 때, 진입점 높이는 불변
- 진입점 높이가 고정점과 보조점 위에 있을 때, 진입점 높이는 고정점과 수평인 높이로 최적화됨
- 진입점과 보조점의 높이가 모두 고정점보다 높을 때, 진입점 높이는 보조점과 수평인 높이로 최적화됨
