## 컨트롤러 IP 설정

### 컨트롤러 IP 설정

**명령어**: 0x4301 `CONTROLLER_IP_SET`

**매개변수 설명**:

| 매개변수 | 유형 | 설명 |
|------|------|------|
| name | string | 네트워크 카드 이름 |
| address | string | IP 주소 |
| gateway | string | 게이트웨이 |
| dns | string | DNS 서버 |

```json
{
  "name": "p1p1",
  "address": "192.168.1.13",
  "gateway": "192.168.1.1",
  "dns": "114.114.114.114"
}
```

### 컨트롤러 IP 조회

**명령어**: 0x4302 `CONTROLLER_IP_INQUIRE`

매개변수 없음

### 컨트롤러 응답

**명령어**: 0x4303 `CONTROLLER_IP_RESPOND`

**매개변수 설명**:

| 매개변수 | 유형 | 설명 |
|------|------|------|
| num | int | 네트워크 카드 수량 |
| network | array | 네트워크 카드 정보 배열 |

```json
{
  "num": 1,
  "network": [
    {
      "name": "p1p1",
      "address": "192.168.1.13",
      "gateway": "192.168.1.1",
      "dns": "114.114.114.114"
    },
    {
      "name": "p2p1",
      "address": "192.168.1.14",
      "gateway": "192.168.1.1",
      "dns": "114.114.114.114"
    }
  ]
}
```

**명령어**: 0x5052 `IDENTIFY_NUMBER_INQUIRE`

| 매개변수 | 유형 | 설명 |

{

}

**명령어**: 0x5055 `USE_REST_DAYS_INQUIRE`

### 컨트롤러 응답

**명령어**: 0x5056 `USE_REST_DAYS_RESPOND`

## 버전 번호 가져오기

티치 펜던트가 현재 컨트롤러 버전 번호를 가져와야 할 때, 다음 명령을 전송합니다:

**명령어**: 0x3402 `VERSIONNUM_INQUIRE`

**매개변수 설명**:

| 매개변수 | 유형 | 설명 |
|------|------|------|
| version | string | 버전 번호 내용 |
| servoVersion | string | 서보 버전 |
| ocmVersion | string | 정적 라이브러리 버전 |
| ioVersion | string | IO 보드 버전 |

```json
{
  "version": "v1.0-rc1-67-gf34dae7",
  "servoVersion": xx,
  "ocmVersion": xx,
  "ioVersion": xx
}
```

### 컨트롤러가 조회 명령을 수신하면, 다음 명령을 전송합니다

**명령어**: 0x3403 `VERSIONNUM_RESPOND`

**매개변수 설명**:

| 매개변수 | 유형 | 설명 |
|------|------|------|
| configFileVersionMismatch | bool | 구성파일 일치 여부, false: 불일치, true: 일치 |
| jobFileVersion | string | 작업파일 버전 |
| rtlVersion | string | 컨트롤러 버전 |
| sysClock | string | 시스템 시계, 형식: YYYY.MM.DD HH:MM:SS |
| version | string | 버전 번호 내용 |
| servoVersion | string | 서보 버전 (구형 컨트롤러는 빈 문자열 응답) |
| ocmVersion | string | 정적 라이브러리 버전 (구형 컨트롤러는 빈 문자열 응답) |
| ioVersion | string | IO 버전 번호 (구형 컨트롤러는 빈 문자열 응답) |
| nodka_c1201 | object | C1201 컨트롤러 전용 노드 |

```json
{
  "configFileVersionMismatch": false,
  "jobFileVersion": "5.2.0",
  "rtlVersion": "22.08.0",
  "sysClock": "2023.11.10 13:29:25",
  "version": "v1.0-rc1-67-gf34dae7",
  "servoVersion": "",
  "ocmVersion": "",
  "ioVersion": "",
  "nodka_c1201": {
    "deviceId": "",
    "hw_version": "",
    "fw_version": "",
    "drv_version": ""
  }
}
```

**nodka_c1201 노드 매개변수 설명**:

| 매개변수 | 유형 | 설명 |
|------|------|------|
| deviceId | string | 장치 ID |
| hw_version | string | 하드웨어 버전 번호 |
| fw_version | string | 펌웨어 버전 번호 |
| drv_version | string | 드라이버 버전 번호 |

## 버전 검증

### 현재 버전의 티치 펜던트 프로그램 존재 여부 조회

**명령어**: 0x3404 `EXIST_TEACHBOX_EQUAL_QTTP_INQUIRE`

**매개변수 설명**:

| 매개변수 | 유형 | 설명 |
|------|------|------|
| MD5 | string | 티치 펜던트 프로그램의 MD5 값 |

```json
{
  "MD5": "xx"
}
```

### 조회 결과 반환

**명령어**: 0x3405 `EXIST_TEACHBOX_EQUAL_QTTP_RESPOND`

조회되지 않은 경우, 현재 버전의 프로그램을 업로드해야 합니다.

**매개변수 설명**:

| 매개변수 | 유형 | 설명 |
|------|------|------|
| backuppath | string | 티치 펜던트 프로그램 경로 |

```json
{
  "backuppath": "xx"
}
```

### 컨트롤러 버전과 일치하는 티치 펜던트 프로그램 존재 여부 조회

**명령어**: 0x3406 `EXIST_CONTROLLER_MATCHED_QTTP_INQUIRE`

매개변수 없음

### 조회 결과 반환

**명령어**: 0x3407 `EXIST_CONTROLLER_MATCHED_QTTP_RESPOND`

결과가 참이면, 팝업 창이 나타나 고객이 자동 업그레이드 여부를 선택하도록 합니다.

**매개변수 설명**:

| 매개변수 | 유형 | 설명 |
|------|------|------|
| backuppath | string | 티치 펜던트 프로그램 경로 |

```json
{
  "backuppath": "xx"
}
```
