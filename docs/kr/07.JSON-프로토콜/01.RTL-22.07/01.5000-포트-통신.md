# 5000 포트 통신(2207)

## 설명

컨트롤러는 5000 포트를 사용하여 파일 전송을 수행하며, 주로 작업 파일 및 구성 파일의 업로드, 다운로드, 백업에 사용됩니다.

### 업로드 프로세스

컨트롤러에 파일을 업로드하려면 상위컴퓨터가 `0x1D00` 요청으로 파일 업로드를 요청하고, 컨트롤러는 `0x1D01` 응답으로 업로드 요청에 답합니다. 컨트롤러가 업로드에 동의하면 상위컴퓨터는 `0x1D04`로 파일을 업로드하고, 업로드 완료 후 상위컴퓨터는 `0x1D05`로 컨트롤러에게 업로드 종료를 알리고, 컨트롤러는 `0x1D06`으로 수신 확인 성공 응답을 회신합니다.

### 다운로드 프로세스

컨트롤러에서 파일을 다운로드하려면 상위컴퓨터가 `0x1D02` 요청으로 파일 다운로드를 요청하고, 컨트롤러는 `0x1D03` 응답으로 다운로드 요청에 답합니다. 컨트롤러가 다운로드에 동의하면 컨트롤러는 `0x1D07`로 다운로드 요청된 파일을 상위컴퓨터로 전송하고, 상위컴퓨터는 `0x1D08`로 컨트롤러에 수신 결과를 회신하며, 다운로드 완료 후 컨트롤러는 `0x1D09`로 상위컴퓨터에게 다운로드 종료를 알리고, 상위컴퓨터는 `0x1D0A`로 수신 확인 성공 응답을 회신합니다.

## 상세 명령 설명

### 0x1D00 상위컴퓨터 파일 업로드 요청

**방향:** 상위컴퓨터 → 컨트롤러

**파라미터 목록:**

| 파라미터명 | 타입 | 설명 |
|--------|------|------|
| name | string | 상위컴퓨터가 보내는 파일의 이름 |
| size | int | 상위컴퓨터가 보내는 파일의 크기 |

**요청 예시:**

```json
{
  "name": "vxWorks",
  "size": 4096
}
```

---

### 0x1D01 컨트롤러 업로드 요청 응답

**방향:** 컨트롤러 → 상위컴퓨터

**파라미터 목록:**

| 파라미터명 | 타입 | 설명 |
|--------|------|------|
| answer | string | 파일 업로드 요청 동의 여부 |

**answer 값 설명:**

| 값 | 설명 |
|----|------|
| yes | 파일 업로드 요청 동의 |
| busy | 컨트롤러 사용 중 |

**응답 예시:**

```json
{
  "answer": "yes"
}
```

---

### 0x1D02 상위컴퓨터 파일 다운로드 요청

**방향:** 상위컴퓨터 → 컨트롤러

**파라미터 목록:**

| 파라미터명 | 타입 | 설명 |
|--------|------|------|
| name | string | 다운로드할 작업 파일 이름 |

**설명:** 파라미터 name은 컨트롤러 내 작업 파일의 경로와 이름을 결합한 것입니다. 0x5533 및 0x5534에서 시스템이 상위컴퓨터로 경로와 이름을 전송합니다. 티치 펜던트가 작업 파일 동기화를 시작하도록 안내합니다.

**요청 예시:**

```json
{
  "name": "job/R1/WWWWW.JBR"
}
```

---

### 0x1D03 컨트롤러 다운로드 요청 응답

**방향:** 컨트롤러 → 상위컴퓨터

**파라미터 목록:**

| 파라미터명 | 타입 | 설명 |
|--------|------|------|
| answer | string | 시스템이 파일 다운로드 요청에 동의하는지 여부 |
| name | string | 다운로드할 작업 파일의 이름 |
| size | int | 이 작업 파일의 크기 |

**answer 값 설명:**

| 값 | 설명 |
|----|------|
| yes | 파일 다운로드 요청 동의 |
| busy | 컨트롤러 사용 중 |

**응답 예시:**

```json
{
  "answer": "yes",
  "name": "job/R1/WWWWW.JBR",
  "size": 217
}
```

### 0x1D06 컨트롤러 수신 확인 성공 응답

**방향:** 컨트롤러 → 상위컴퓨터

**파라미터 목록:**

| 파라미터명 | 타입 | 설명 |
|--------|------|------|
| finish | bool | 파일 업로드 성공 여부 |

**finish 값 설명:**

| 값 | 설명 |
|----|------|
| true | 업로드 성공 |
| false | 업로드 실패 |

**응답 예시:**

```json

  "finish": true
```
