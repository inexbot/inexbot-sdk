# 시스템

## 상태

### 초기화 상태 조회

시스템 시동 시 초기화가 필요하며, 그렇지 않으면 시스템을 사용할 수 없습니다.

**명령어:** `0x1000`

- 전송할 데이터 없음

**컨트롤러 응답**

**명령어:** `0x1001`

| 필드 | 유형 | 설명 |
|------|------|------|
| finishinit | bool | 초기화 완료 여부, true 완료, false 미완료 |

```json
{
  "finishinit": true
}
```

### T5 시스템 여부 조회

**명령어:** `0x1002`

- 전송할 데이터 없음

**컨트롤러 응답**

**명령어:** `0x1003`

| 필드 | 유형 | 설명 |
|------|------|------|
| T5_system | bool | T5 시스템 여부, true 예, false 아니오 |

```json
{
  "T5_system": false
}
```

### 하드웨어 시스템 릴리스 버전 조회

**명령어:** `0x1004`

- 전송할 데이터 없음

**컨트롤러 응답**

**명령어:** `0x1005`

| 필드 | 유형 | 설명 |
|------|------|------|
| result | bool | 조회 성공 여부, true 성공, false 실패 |
| version | string | 버전 번호 (조회 성공 시 반환) |

```json
{
  "result": true,

```

## 버전 업그레이드

시스템 시동 시 현재 버전 번호를 조회해야 하며, 불일치가 발생하면 버전 업그레이드가 필요합니다.

### 현재 시스템 버전 번호 조회

**명령어:** `0x1010`

| 필드 | 유형 | 설명 |
|------|------|------|
| version | string | 버전 번호 내용 |

```json
{
  "version": "v1.0-rc1-67-gf34dae7"
}
```

**컨트롤러 응답**

**명령어:** `0x1011`

| 필드 | 유형 | 설명 |
|------|------|------|
| configFileVersionMismatch | bool | 구성 파일 일치 여부, false 불일치, true 일치 |
| jobFileVersion | string | 작업 파일 버전 |
| rtlVersion | string | 컨트롤러 버전 |
| sysClock | string | 시스템 시계, 형식은 "2023.11.10 13:29:25" |
| version | string | 버전 번호 내용, 조회 시 전송된 version이 빈 문자열이면 동일한 내용 반환 |

```json
{
  "configFileVersionMismatch": false,
  "jobFileVersion": "5.2.0",
  "rtlVersion": "22.08.0",
  "sysClock": "2023.11.10 13:29:25",
  "version": "v1.0-rc1-67-gf34dae7"
}
```

|------|------|------|
| num | int | 최근 몇 개 파일을 가져올지 표시, 5, 30, 100 가능 |

```json
  "num": 5
```

**컨트롤러 응답**

**명령어:** `0x1023`

| 필드 | 유형 | 설명 |
|------|------|------|
| absolutepath | string | 로그가 있는 디렉터리 |
| logfilenum | int | 로그 개수 (num과 반드시 같지는 않음) |
| logfilelist | string[] | 로그 목록 |

```json
  "absolutepath": "log/",
  "logfilenum": 3,
```


## IP 설정

### 컨트롤러 IP 설정

**명령어:** `0x1040`

| 필드 | 유형 | 설명 |
|------|------|------|
| name | string | 네트워크 포트 이름 |
| address | string | 설정할 컨트롤러 IP |
| gateway | string | 설정할 컨트롤러 게이트웨이 |
| dns | string | 설정할 컨트롤러 DNS |

```json
{
  "name": "p1p1",
  "address": "192.168.1.13",
  "gateway": "192.168.1.1",
  "dns": "114.114.114.114"
}
```

### 연결 IP 조회

**명령어:** `0x1041`

- 전송할 데이터 없음

**컨트롤러 응답**

**명령어:** `0x1042`

| 필드 | 유형 | 설명 |
|------|------|------|
| num | int | 네트워크 포트 개수 |
| network | array | 네트워크 정보 목록 |
| network[].name | string | 네트워크 포트 이름 |
| network[].address | string | 컨트롤러 IP |
| network[].gateway | string | 컨트롤러 게이트웨이 |
| network[].dns | string | 컨트롤러 DNS |

```json
{
  "num": 2,
  "network": [
    {
      "name": "p1p1",
      "address": "192.168.1.13",
      "gateway": "192.168.1.1",
      "dns": "114.114.114.114"
    },
    {
      "name": "p2p1",
      "address": "192.168.1.14",
      "gateway": "192.168.1.1",
      "dns": "114.114.114.114"
    }
  ]
}
```

## 시간 설정

### 시간 조회 전송

**명령어:** `0x1050`

| 필드 | 유형 | 설명 |
|------|------|------|
| type | int | 시간 유형: 0 실시간 시간, 1 시스템 시작 시간부터 현재까지 s 수 |
| format | int | 형식: 0 타임스탬프, 1 베이징 시간 |

```json
{
  "type": 0,
  "format": 0
}
```

**컨트롤러 응답**

**명령어:** `0x1051`

| 필드 | 유형 | 설명 |
|------|------|------|
| tType | int | 시간 유형: 0 실시간 시간, 1 시스템 시작 시간부터 현재까지 s 수 |
| format | int | 형식: 0 타임스탬프, 1 베이징 시간 |
| date | object | 날짜 및 시간 정보 |
| date.year | int | 연 |
| date.mon | int | 월 |
| date.day | int | 일 |
| date.hour | int | 시 |
| date.min | int | 분 |
| date.sec | int | 초 |
| date.msec | int | 밀리초 |
| timestamp | object | 타임스탬프 (format이 0일 때 표시) |
| timestamp.s | int | 타임스탬프 초 수 |
| timestamp.ns | int | 타임스탬프 나노초 수 |

```json
{
  "tType": 0,
  "format": 0,
  "date": {
    "year": 2023,
    "mon": 1,
    "day": 1,
    "hour": 1,
    "min": 1,
    "sec": 1,
    "msec": 1
  },
  "timestamp": {
    "s": 1,
    "ns": 1
  }
}
```

## 시스템 백업

### 현재 버전 티치 펜던트 프로그램 존재 여부 조회

**명령어:** `0x1090`

| 필드 | 유형 | 설명 |
|------|------|------|
| config_md5 | string | config.db 파일 MD5 값 |
| pinyin_md5 | string | Pinyin.db 파일 MD5 값, null은 파일이 없거나 크기가 0임을 표시 |
| pro_md5 | string | 티치 펜던트 프로그램 파일의 MD5 값 |
| pro_name | string | 티치 펜던트 프로그램 파일의 이름, Qt-tp 또는 T30_ARM_Linux_Teach-pendant 선택 |

```json
{
  "config_md5": "2803f92ecdecfd3c178b0998b9391a4d",
  "pinyin_md5": "null",
  "pro_md5": "ce407851346d030dad5207948f55ed6c",
  "pro_name": "Qt-tp"
}
```

**컨트롤러 응답**

**명령어:** `0x1091`

| 필드 | 유형 | 설명 |
|------|------|------|
| path | object | 파일 경로 정보 |
| path.config | string | config.db 파일 업로드 경로 |
| path.dir | string | 모든 파일의 경로 |
| path.pinyin | string | Pinyin.db 파일 업로드 경로, 빈 문자열은 업로드 불필요 표시 |
| path.pro | string | Qt-tp 파일 업로드 경로 |

```json
{
  "path": {
    "config": "qttpBackup/rtl-24.03.07-6.0.9-1.0.6",
    "dir": "qttpBackup/rtl-24.03.07-6.0.9-1.0.6",

    "pro": "qttpBackup/rtl-24.03.07-6.0.9-1.0.6"

}
```

### 컨트롤러 ID 조회

**명령어:** `0x10B3`

- 전송할 데이터 없음

**컨트롤러 응답**

**명령어:** `0x10B4`

| 필드 | 유형 | 설명 |
|------|------|------|
| controllerID | string | 컨트롤러 ID |

```json
{
  "controllerID": "4DC9F0249098C82E"
}
```

> 설명: 연결된 컨트롤러 ID를 가져온 후, 작업 파일 동기화 시 이 ID가 알려진 ID인지 판단해야 합니다. 알려진 경우 경로 "dir/controllerID" 존재 여부를 판단하고, 존재하면 작업하지 않음; 존재하지 않으면 작업 파일 경로를 생성하며, 형식은 "dir/controllerID/robotJob/R1". 여러 로봇은 R1-4로 표시합니다.
