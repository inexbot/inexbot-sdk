# 티치 펜던트와 호스트 컴퓨터의 통신 연결

## 하트비트

> **설명**: 어떤 통신 모드든 하트비트 방식을 설계하여 통신 상태를 정상적으로 유지해야 합니다. 티치 펜던트는 정기적으로 컨트롤러에 하트비트 확인 패킷을 보내고 컨트롤러의 응답을 기다립니다. 클라이언트는 하트비트 패킷을 받은 후 하트비트 응답 패킷을 회신해야 하며, 패킷 데이터는 확인 패킷에 포함된 타임스탬프입니다. 티치 펜던트는 응답 패킷을 받은 후 데이터가 올바른지 확인합니다. 데이터가 올바르지 않거나 응답을 받지 못한 경우 연결이 끊어졌을 가능성이 있다고 간주합니다.

### 하트비트 수신

| 속성 | 유형 | 설명 |
|------|------|------|
| time | int | 타임스탬프 |

**명령어**: `0x1200`

```json
{
  "time": 123
}
```

### 하트비트 응답

| 속성 | 유형 | 설명 |
|------|------|------|
| time | int | 타임스탬프 |

**명령어**: `0x1201`

```json
{
  "time": 123
}
```
