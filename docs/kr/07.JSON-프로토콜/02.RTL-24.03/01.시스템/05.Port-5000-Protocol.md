# 5000포트协议

## 설명

5000포트协议主要作用于上传、다운로드、备份파일。

- 若要向컨트롤러上传파일，则호스트 컴퓨터通过0x1D00요청上传파일，컨트롤러通过0x1D01回复上传요청，当컨트롤러同意上传时，호스트 컴퓨터通过0x1D04上传파일，上传完毕后，호스트 컴퓨터通过0x1D05告诉컨트롤러上传结束，컨트롤러通过0x1D06回复确认수신成功。
- 若要다운로드컨트롤러中的파일，则호스트 컴퓨터通过0x1D02요청다운로드파일，컨트롤러通过0x1D03回复다운로드요청，当컨트롤러同意다운로드时，컨트롤러通过0x1D07向호스트 컴퓨터传输其申请다운로드的파일，并且호스트 컴퓨터通过0x1D08回复컨트롤러수신结果，다운로드完毕后，컨트롤러通过0x1D09告诉호스트 컴퓨터다운로드结束，호스트 컴퓨터通过0x1D0A回复确认수신成功。

## 详细명령설명

### 0x1D00 호스트 컴퓨터요청上传파일

**方向：** 호스트 컴퓨터 → 컨트롤러

**매개변수列表：**

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| name | string | 호스트 컴퓨터 전송파일的파일名 |
| size | int | 호스트 컴퓨터 전송파일的파일大小 |

**요청예시：**

```json
{
  "name": "vxWorks",
  "size": 4096
}
```

---

### 0x1D01 컨트롤러 응답上传요청

**方向：** 컨트롤러 → 호스트 컴퓨터

**매개변수列表：**

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| answer | string | 是否同意上传파일요청 |

**answer 值설명：**

| 值 | 설명 |
|----|------|
| yes | 同意上传파일요청 |
| busy | 컨트롤러忙碌 |

**응답예시：**

```json
{
  "answer": "yes"
}
```

---

### 0x1D02 호스트 컴퓨터요청다운로드파일

**方向：** 호스트 컴퓨터 → 컨트롤러

**매개변수列表：**

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| name | string | 필요다운로드的작업파일名称 |

**설명：** 매개변수name是작업파일在컨트롤러中的작업파일路径加上名字。在0x5533和0x5534中시스템有向호스트 컴퓨터 전송路径和名字。티치 펜던트提示开始同步작업파일。

**요청예시：**

```json
{
  "name": "job/R1/WWWWW.JBR"
}
```

---

### 0x1D03 컨트롤러 응답다운로드요청

**方向：** 컨트롤러 → 호스트 컴퓨터

**매개변수列表：**

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| answer | string | 시스템是否同意다운로드파일요청 |
| name | string | 필요다운로드的작업파일名称 |
| size | int | 此작업파일的파일大小 |

**answer 值설명：**

| 值 | 설명 |
|----|------|
| yes | 同意다운로드파일요청 |
| busy | 컨트롤러忙碌 |

**응답예시：**

```json
{
  "answer": "yes",
  "name": "job/R1/WWWWW.JBR",
  "size": 217
}
```

---

### 0x1D04 호스트 컴퓨터向컨트롤러上传파일

**方向：** 호스트 컴퓨터 → 컨트롤러

**설명：** 上传파일데이터

---

### 0x1D05 호스트 컴퓨터告诉컨트롤러上传结束

**方向：** 호스트 컴퓨터 → 컨트롤러

---

### 0x1D06 컨트롤러 응답确认수신成功

**方向：** 컨트롤러 → 호스트 컴퓨터

**매개변수列表：**

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| finish | bool | 判断파일是否上传成功 |

**finish 值설명：**

| 值 | 설명 |
|----|------|
| true | 上传成功 |
| false | 上传失败 |

**응답예시：**

```json
{
  "finish": true
}
```

---

### 0x1D07 컨트롤러向호스트 컴퓨터 전송申请다운로드的파일

**方向：** 컨트롤러 → 호스트 컴퓨터

**설명：** 传输다운로드파일데이터

---

### 0x1D08 호스트 컴퓨터回复수신结果

**方向：** 호스트 컴퓨터 → 컨트롤러

---

### 0x1D09 컨트롤러告诉호스트 컴퓨터다운로드完成

**方向：** 컨트롤러 → 호스트 컴퓨터

---

### 0x1D0A 호스트 컴퓨터回复确认수신成功

**方向：** 호스트 컴퓨터 → 컨트롤러

**매개변수列表：**

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| finish | bool | 判断파일是否다운로드成功 |

**finish 值설명：**

| 值 | 설명 |
|----|------|
| true | 다운로드成功 |
| false | 다운로드失败 |

**응답예시：**

```json
{
  "finish": true
}
```

---

### 0x1D0B 다운로드연결断开

**설명：** 断开다운로드연결

---

### 0x1D10 호스트 컴퓨터요청备份파일

**方向：** 호스트 컴퓨터 → 컨트롤러

---

### 0x1D11 컨트롤러 응답备份요청

**方向：** 컨트롤러 → 호스트 컴퓨터

**매개변수列表：**

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| answer | string | 回复备份파일名 |

**응답예시：**

```json
{
  "answer": "controller.tar.gz"
}
```

---

### 0x1D12 호스트 컴퓨터 전송다운로드log파일完成

**方向：** 호스트 컴퓨터 → 컨트롤러

---

### 0x1D13 호스트 컴퓨터 전송다운로드eni파일完成

**方向：** 호스트 컴퓨터 → 컨트롤러

### 호스트 컴퓨터回复确认수신成功

**명령字：** `0x1D0a`

| 필드 | 유형 | 설명 |
|------|------|------|
| finish | bool | 判断파일是否다운로드成功，true 다운로드成功，false 다운로드失败 |

```json
{
  "finish": true
}
```

### 다운로드연결断开

**명령字：** `0x1D0b`

- 호스트 컴퓨터요청备份파일
