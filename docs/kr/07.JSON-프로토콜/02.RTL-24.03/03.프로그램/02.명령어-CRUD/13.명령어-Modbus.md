## MODBUS_OPEN - 열기Modbus연결

### 설명

열기Modbus통신연결

### 매개변수 목록

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| ID | int | 예 | Modbus主站프로세스号，取值범위[1,9] |
| logout | bool | 아니오 | 주석 처리 여부。true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| modType | string | 예 | 主站/从站。"Master"：主站；"Slave"：从站 |
| type | int | 예 | MODBUS_OPEN在열거형数열type中为210 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |

### 예시

```json
{
  "ID": 5,
  "logout": false,
  "modType": "Master",
  "type": 210,
  "userParamInt": 0,
  "userParamString": ""
}
```

## MODBUS_READ - Modbus读상태

### 설명

읽기Modbus中对应주소的값。将가져오기到的상태存入到변수（bool유형변수或绑定변수，全局或局部）。

**주소유형설명：**

| 주소유형 | 설명 |
|----------|------|
| 3x | 읽기Modbus中对应주소的值 |
| 4x | 同3x |
| 3x-bit | 읽기所选주소中所存值得任一bit的상태 |
| 4x-bit | 同3x-bit |
| 0x | 可读쓰기 가능的设备유형，相当于조작PLC的출력点 |
| 1x | 읽기 전용设备유형，相当于조작PLC的입력点 |

**주소유형为3X-bit时从站주소번째数범위예[0,16]**

**주소유형为3X-bit，存到변수时候的17비트为符号비트，0表示正数，1表示负数**

**읽기字符时会将읽기到的所有值存入到一번째변수，它跟정수형、부동소수점형、불리언형변수不一样，不会根据설정的首주소，读从站주소번째数顺延存入其它변수**

### 매개변수 목록

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| ID | int | 예 | Modbus主站프로세스号，取值범위[1,9] |
| addressNumber | object | 예 | 读从站주소번째数。手填或者int유형변수或绑定변수，全局或局部。읽기的주소번째数，如果설정的读从站주소번째数为3，실행读명령어后会从Modbus首주소시작읽기3번째주소（包含首주소一共读3번째） |
| addressType | string | 예 | 주소유형。선택取值：3x，4x，3x-bit，4x-bit，0x，1x |
| dataType | string | 예 | 데이터유형。当데이터存放的首번째변수选I或GI时，从站的데이터유형선택short和ushort（分别对应短정수형和없음符号短정수형，前者범위为[-32768,32767]，后者的범위为[0,65535]）。데이터存放首번째변수选I或GI데이터유형时需要和从站的데이터유형一致，아니오则可能导致데이터出错 |
| dataVariable | object | 예 | 데이터存放的首번째변수。所有변수或绑定변수都可。将읽기到的매개변수存放到변수，如果选择的首번째변수为I001，읽기的주소번째数为3，실행读명령어会把매개변수值依회存入到I001、I002、I003 |
| firstAddress | object | 예 | 从站레지스터首주소。手填或int유형변수或绑定변수，全局或局部。根据명령어매개변수里面설정的首주소읽기매개변수值，如果매개변수里面설정的首주소예5，在실행读명령어后会从Modbus제5번째주소시작读 |
| logout | bool | 아니오 | 주석 처리 여부。true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| modType | string | 예 | 主站/从站。"Master"：主站；"Slave"：从站 |
| type | int | 예 | MODBUS_READ在열거형数열type中为213 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |

### 예시

```json
{
  "ID": 3,
  "addressNumber": {"data": 0.0, "secondvalue": 1, "value": 4, "varname": "GI[I001]"},
  "addressType": "4x-bit",
  "dataType": "short",
  "dataVariable": {"data": 0.0, "secondvalue": 1, "value": 9, "varname": "GS[GI001]"},
  "firstAddress": {"data": 0.0, "secondvalue": 1, "value": 4, "varname": "GI[GI001]"},
  "logout": false,
  "modType": "Master",
  "type": 213,
  "userParamInt": 0,
  "userParamString": ""
}
```

## MODBUS_WRITE - Modbus写조작

### 설명

该명령어用于通过Modbus将변수쓰기从站레지스터中对应위치的주소中

**주소유형설명：**

| 주소유형 | 설명 |
|----------|------|
| 4x | 将변수里的值쓰기주소码 |
| 4x-bit | 将변수组成16비트二进制码쓰기주소码中 |
| 0x | 将변수值쓰기주소码，只有"0"，"1"两种상태，변수有值的话쓰기1，변수没值쓰기0 |

### 매개변수 목록

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| ID | int | 예 | Modbus主站프로세스号，取值범위[1,9] |
| addressNumber | object | 예 | 读从站주소번째数。手填或者int유형변수或绑定변수，全局或局部。읽기的주소번째数，如果설정的读从站주소번째数为3，실행读명령어后会从Modbus首주소시작읽기3번째주소（包含首주소一共读3번째） |
| addressType | string | 예 | 주소유형。선택取值：4x，4x-bit，0x |
| dataVariable | object | 예 | 데이터存放的首번째변수，쓰기的변수유형。手填或者所有변수或绑定변수都可 |
| firstAddress | object | 예 | 从站레지스터首주소。手填或int유형변수或绑定변수，全局或局部。시작쓰기的제一번째주소，如果首주소为5，실행写명령어会从modbus제5번째주소시작쓰기 |
| logout | bool | 아니오 | 주석 처리 여부。true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| modType | string | 예 | 主站/从站。"Master"：主站；"Slave"：从站 |
| type | int | 예 | MODBUS_WRITE在열거형数열type中为214 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |

### 예시

```json
{
  "ID": 5,
  "addressNumber": {"data": 0.0, "secondvalue": 0, "value": 1, "varname": "I001"},
  "addressType": "0x",
  "dataVariable": {"data": 0.0, "secondvalue": 1, "value": 4, "varname": "GI[GI001]"},
  "firstAddress": {"data": 0.0, "secondvalue": 1, "value": 1, "varname": "I[GI001]"},
  "logout": false,
  "modType": "Master",
  "type": 214,
  "userParamInt": 0,
  "userParamString": ""
}
```
