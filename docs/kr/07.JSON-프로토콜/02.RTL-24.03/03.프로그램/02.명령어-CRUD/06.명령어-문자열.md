# 명령어-字符串类

---

## 1. 字符串追加 (STRING_SPELL)

**설명**：在原有字符串변수或者新的字符변수中追加需要的字符，就构成了一个新的字符串변수。

**명령어名**：STRING_SPELL

**type 值**：223

### 매개변수 목록

| 매개변수名 | 유형 | 설명 | 예시值 |
|--------|------|------|--------|
| logout | bool | 주석 처리 여부。true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 | false |
| type | int | 명령어유형，字符串追加在枚举数列中为 223 | 223 |
| userParamInt | int | 2차 개발에서 사용자 정의, 현재 의미 없음 | 0 |
| userParamString | string | 2차 개발에서 사용자 정의, 현재 의미 없음 | "" |
| variable | object | 변수유형及名称；string유형변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"S001"}` |
| variableValue | object | 追加的字符변수；string유형변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"GS001"}` |

### JSON 예시

```json
{
  "logout": false,
  "type": 223,
  "userParamInt": 0,
  "userParamString": "",
  "variable": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "S001"},
  "variableValue": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "GS001"}
}
```

## 2. 索引截取 (STRING_SLICE)

**설명**：截取一个字符串변수中的其中一部分字符串，并把截取到部分字符串存到指定的변수。

**명령어名**：STRING_SLICE

**type 值**：224

### 매개변수 목록

| 매개변수名 | 유형 | 설명 | 예시值 |
|--------|------|------|--------|
| logout | bool | 주석 처리 여부。true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 | false |
| type | int | 명령어유형，索引截取在枚举数列中为 224 | 224 |
| userParamInt | int | 2차 개발에서 사용자 정의, 현재 의미 없음 | 0 |
| userParamString | string | 2차 개발에서 사용자 정의, 현재 의미 없음 | "" |
| startIndex | object | 起始索引위치；int유형手填值或者변수，全局或局部 | `{"data":2.0,"secondvalue":0,"value":0,"varname":""}` |
| endIndex | object | 结束索引위치；int유형手填值或者변수，全局或局部 | `{"data":8.0,"secondvalue":0,"value":0,"varname":""}` |
| dataVariable | object | 조회的데이터依次存放的위치；string유형변수，全局或局部 | `{"data":0.0,"secondvalue":1,"value":8,"varname":"GS[I001]"}` |
| variable | object | 被截取的字符串변수；string유형변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"S001"}` |

### JSON 예시

```json
{
  "logout": false,
  "type": 224,
  "userParamInt": 0,
  "userParamString": "",
  "startIndex": {"data": 2.0, "secondvalue": 0, "value": 0, "varname": ""},
  "endIndex": {"data": 8.0, "secondvalue": 0, "value": 0, "varname": ""},
  "dataVariable": {"data": 0.0, "secondvalue": 1, "value": 8, "varname": "GS[I001]"},
  "variable": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "S001"}
}
```

## 3. 字符串分隔符拆分 (STRING_SPLIT)

**설명**：将字符串변수中的其中一个字符拆分，并把拆分的字符依次存放到指定변수中。

**명령어名**：STRING_SPLIT

**type 值**：225

### 매개변수 목록

| 매개변수名 | 유형 | 설명 | 예시值 |
|--------|------|------|--------|
| logout | bool | 주석 처리 여부。true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 | false |
| type | int | 명령어유형，字符串分隔符拆分在枚举数列中为 225 | 225 |
| userParamInt | int | 2차 개발에서 사용자 정의, 현재 의미 없음 | 0 |
| userParamString | string | 2차 개발에서 사용자 정의, 현재 의미 없음 | "" |
| separator | object | 分隔符；string유형的手填值或변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"GS001"}` |
| variable | object | 被提取的字符串변수名；string유형변수，全局或局部 | `{"data":0.0,"secondvalue":1,"value":8,"varname":"S[I001]"}` |
| dataVariable | object | 조회的데이터依次存放的首위치；string유형변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"S001"}` |
| dataStorage | object | 记录提取데이터的数量；string유형的手填"0"或者int유형변수，全局或局部。string유형的手填"0"表示不使用 | `{"data":0.0,"secondvalue":1,"value":8,"varname":"GI[I001]"}` |

### JSON 예시

```json
{
  "dataStorage": {"data": 0.0, "secondvalue": 1, "value": 8, "varname": "GI[I001]"},
  "dataVariable": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "S001"},
  "logout": false,
  "separator": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "GS001"},
  "type": 225,
  "userParamInt": 0,
  "userParamString": "",
  "variable": {"data": 0.0, "secondvalue": 1, "value": 8, "varname": "S[I001]"}
}
```

## 4. 字符串定位 (STRING_LOCATE)

**설명**：조회字符串변수里面的一种字符所在的위치，并把위치和数量存到指定변수。

> **注意**：
> - 如果待索引변수在一个字符串里面是不连续的字符，在执行定位조회명령어时，字符的위치和个数是无法读取出来的
> - 如果待索引변수在一个字符串里面是连续的字符，在执行定位조회명령어时，读取出来的字符위치是第一个索引변수的위치，데이터存放个数为1

**명령어名**：STRING_LOCATE

**type 值**：226

### 매개변수 목록

| 매개변수名 | 유형 | 설명 | 예시值 |
|--------|------|------|--------|
| logout | bool | 주석 처리 여부。true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 | false |
| type | int | 명령어유형，字符串定位在枚举数列中为 226 | 226 |
| userParamInt | int | 2차 개발에서 사용자 정의, 현재 의미 없음 | 0 |
| userParamString | string | 2차 개발에서 사용자 정의, 현재 의미 없음 | "" |
| variable | object | 被提取的字符串변수名；string유형변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"S001"}` |
| startIndex | object | 需要搜索的字符；string유형手填值或변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":7,"varname":"RRRR"}` |
| dataVariable | object | 조회的데이터依次存放的首위치；int유형변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"I001"}` |
| dataStorage | object | 记录提取데이터的数量；string유형的手填"0"或者int유형변수，全局或局部。string유형的手填"0"表示不使用 | `{"data":0.0,"secondvalue":1,"value":8,"varname":"GI[GI001]"}` |

### JSON 예시

```json
{
  "dataStorage": {"data": 0.0, "secondvalue": 1, "value": 8, "varname": "GI[GI001]"},
  "dataVariable": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "I001"},
  "logout": false,
  "startIndex": {"data": 0.0, "secondvalue": 0, "value": 7, "varname": "RRRR"},
  "type": 226,
  "userParamInt": 0,
  "userParamString": "",
  "variable": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "S001"}
}
```

## 6. 字符串转非字符串 (STRING_TO)

**설명**：字符串변수转换成非字符串변수。

> **转换规则**：字符串변수转换成非字符串변수时只会将字符串변수的数值部分转译到非字符串변수。在转译过程中会从字符串변수的第一个字符开始转译，直到检测到的字符是非数值的字符时转译结束，将检测到的数值部分的字符转译到非字符변수。如果检测到的第一个字符就是非数值的字符，那字符串无法转译成功。

**명령어名**：STRING_TO

**type 值**：228

### 매개변수 목록

| 매개변수名 | 유형 | 설명 | 예시值 |
|--------|------|------|--------|
| logout | bool | 주석 처리 여부。true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 | false |
| type | int | 명령어유형，字符串转非字符串在枚举数列中为 228 | 228 |
| userParamInt | int | 2차 개발에서 사용자 정의, 현재 의미 없음 | 0 |
| userParamString | string | 2차 개발에서 사용자 정의, 현재 의미 없음 | "" |
| variable | object | 待计算长度的변수；string유형변수，全局或局部 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"S001"}` |
| nonVariable | object | 转译的目标변수；int유형或double유형或bool유형변수，局部或全局 | `{"data":0.0,"secondvalue":0,"value":8,"varname":"I001"}` |

### JSON 예시

```json
{
  "nonVariable": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "I001"},
  "logout": false,
  "type": 228,
  "userParamInt": 0,
  "userParamString": "",
  "variable": {"data": 0.0, "secondvalue": 0, "value": 8, "varname": "S001"}
}
```
