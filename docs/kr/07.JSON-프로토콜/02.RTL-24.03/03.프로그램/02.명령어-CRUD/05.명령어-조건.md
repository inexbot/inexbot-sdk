# 명령어-条件제어클래스

## 호출子프로그램 (CALL)

**설명**：호출另一번째프로그램，被호출的프로그램실행完后반환主프로그램CALL명령어的下一줄계속실행。

**명령어名**：CALL

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| callInParaVarData | array | 아니오 | 입력매개변수번째数及选择；有0-12번째元素。元素형식：`{"data":0.0,"secondvalue":0,"value":2,"varname":"D001"}` |
| callOutParaVarData | array | 아니오 | 출력매개변수번째数及选择；有0-12번째元素。元素형식：`{"data":0.0,"secondvalue":0,"value":2,"varname":"D001"}` |
| jobName | string | 예 | 호출프로그램이름，不能예本身，호출的프로그램中须有매개변수声明 |
| logout | bool | 아니오 | 주석 처리 여부；true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| type | int | 예 | CALL在열거형数열中为15 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |

**예시**：有两번째매개변수

```json
{
    "callInParaVarData": [
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 2,
            "varname": "D001"
        },
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 2,
            "varname": "D002"
        }
    ],
    "callOutParaVarData": [
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 5,
            "varname": "GD003"
        },
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 5,
            "varname": "GD004"
        }
    ],
    "jobName": "QQQQ",
    "logout": false,
    "type": 15,
    "userParamInt": 0,
    "userParamString": ""
}
```

## 종료 (RETURN)

**설명**：종료현재실행的작업파일，例如：子프로그램、后台프로그램。

**명령어名**：RETURN

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| logout | bool | 아니오 | 주석 처리 여부；true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| type | int | 예 | RETURN在열거형数열中为251 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |

**예시**：

```json
{
    "logout": false,
    "type": 251,
    "userParamInt": 0,
    "userParamString": ""
}
```

## 호출Lua파일 (CALL_LUAFILE)

**설명**：매개변수定义好之后，直接업로드Lua파일，然后삽입호출Lua파일명령어，명령어실행종료后会把写好的所有매개변수변수传入和출력。

**명령어名**：CALL_LUAFILE

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| logout | bool | 아니오 | 주석 처리 여부；true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| type | int | 예 | CALL_LUAFILE在열거형数열中为218 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| inParaVarData | array | 아니오 | 传入매개변수번째数及选择；元素번째数[0-12]。元素형식：`{"data":0.0,"secondvalue":0,"value":1,"varname":"I001"}` |
| luafileName | string | 예 | 호출的lua파일 |
| outParaVarData | array | 아니오 | 출력매개변수번째数及选择；元素번째数[0-12]。元素형식：`{"data":0.0,"secondvalue":0,"value":1,"varname":"I001"}` |
| textEditType | bool | 아니오 | 文本编译유형；true：文本유형，false：语句유형 |

**예시**：

```json
{
    "logout": false,
    "type": 218,
    "userParamInt": 0,
    "userParamString": "",
    "inParaVarData": [
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 1,
            "varname": "I001"
        },
        {
            "data": 2.0,
            "secondvalue": 0,
            "value": 0,
            "varname": ""
        }
    ],
    "luafileName": "demo.lua",
    "outParaVarData": [
        {
            "data": 3.0,
            "secondvalue": 0,
            "value": 0,
            "varname": ""
        },
        {
            "data": 6.0,
            "secondvalue": 0,
            "value": 0,
            "varname": ""
        }
    ],
    "textEditType": true
}
```

## 如果 (IF)

**설명**：通过判断条件여부成立来실행下一目标的명령어，判断条件不满足时，실행ELSEIF语句。IF和ENDIF配对使用，不能单独삭제或更改，会导致프로그램실행出错。ELSE IF不可脱离IF单独使用。ELSE不可脱离IF单独使用。

**명령어名**：IF

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| logout | bool | 아니오 | 주석 처리 여부；true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| type | int | 예 | IF在열거형数열中为9 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| ifconditionArrayFINAL | object | 예 | 表示if判断；{}内的为if判断的데이터 |

**ifconditionArrayFINAL 结构**：

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| conditionGroup | array | 元素번째数表示有多少번째判断 |
| logic | array | if判断与if判断之间的关系，按배열内元素顺序；1：或，0：与 |
| logicGroup | array | conditionGroup节点的元素的判断条件；1：或，0：与，null表示该层没有判断 |

**conditionGroup 元素结构**：

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| logicType | int | if判断条件比较方式；1：==，2：<，3：>，4：<=，5：>=，6：!= |
| paraGroupOne | object | 변수1，형식：`{"data":0.0,"secondvalue":0,"value":1,"varname":"I002"}` |
| paraGroupTwo | object | 변수2，형식：`{"data":0.0,"secondvalue":0,"value":1,"varname":"I002"}` |

**예시**：

```json
{
    "logout": false,
    "type": 9,
    "userParamInt": 0,
    "userParamString": "",
    "ifconditionArrayFINAL": {
        "conditionGroup": [
            [
                {
                    "logicType": 2,
                    "paraGroupOne": {
                        "data": 0.0,
                        "secondvalue": 0,
                        "value": 1,
                        "varname": "I001"
                    },
                    "paraGroupTwo": {
                        "data": 0.0,
                        "secondvalue": 0,
                        "value": 0,
                        "varname": ""
                    }
                }
            ],
            [
                {
                    "logicType": 1,
                    "paraGroupOne": {
                        "data": 0.0,
                        "secondvalue": 0,
                        "value": 1,
                        "varname": "I001"
                    },
                    "paraGroupTwo": {
                        "data": 0.0,
                        "secondvalue": 0,
                        "value": 0,
                        "varname": ""
                    }
                }
            ],
            [
                {
                    "logicType": 6,
                    "paraGroupOne": {
                        "data": 0.0,
                        "secondvalue": 0,
                        "value": 1,
                        "varname": "I002"
                    },
                    "paraGroupTwo": {
                        "data": 0.0,
                        "secondvalue": 0,
                        "value": 0,
                        "varname": ""
```

## 대기 (WAIT)

**설명**：条件成立之前프로그램处于대기상태，条件成立后계속실행대기명령어后面的명령어。

**명령어名**：WAIT

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| logout | bool | 아니오 | 주석 처리 여부；true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| type | int | 예 | WAIT在열거형数열中为8 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| m_filterTime | double | 아니오 | 滤波时间，입력信号需满足的时间；[0,9999]s。滤波时间要小于대기时间 |
| paraTime | object | 아니오 | 대기时间；double유형；[0.01,9999]s |
| para_wait_result_ | object | 아니오 | 대기结果，满足条件为true；bool유형 |
| waitConditionFinal | object | 아니오 | {}内为参与判断的条件和변수 |
| wait_value_check_ | bool | 아니오 | 여부接受结果；true：존재"para_wait_result_"节点，false：존재하지 않음 |

## WHILE 循环 (WHILE)

**설명**：判断条件为真예反复실행循环语句里面的명령어。삽입WHILE명령어的同时会同时삽입ENDWHILE명령어。若要삭제WHILE명령어请同时删掉其对应的ENDWHILE명령어，아니오则会导致프로그램없음法실행。当프로그램的开头为WHILE且最后一样명령어为ENDWHILE时，请在프로그램的开头或结尾삽입一条TIMER（延时）명령어。아니오则当WHILE명령어的条件不满足时会导致프로그램陷入死机。当WHILE内部的명령어没有모션클래스명령어或在某种情况下可能会陷入死循环时，请在WHILE与ENDWHILE间삽입一条TIMER（延时）명령어，아니오则当WHILE명령어的条件满足时可能会导致프로그램陷入死机。当WHILE명령어的条件满足时，会循环실행WHILE与ENDWHILE两条명령어之间的명령어。在실행到WHILE명령어之前若判断条件不满足，在실행到WHILE명령어时会直接跳转到ENDWHILE명령어而不실행WHILE与ENDWHILE之间的명령어；若在실행WHILE与ENDWHILE之间的명령어过程中，判断条件变成不满足，会계속실행，直到실행到ENDWHILE줄，不再循环而예계속실행ENDWHILE下面的명령어。

**명령어名**：WHILE

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| logout | bool | 아니오 | 주석 처리 여부；true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| type | int | 예 | WHILE在열거형数열中为13 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| whileconditionArrayFINAL | object | 예 | 表示while判断；{}内的为while判断的데이터 |

**whileconditionArrayFINAL 结构**：

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| conditionGroup | array | 元素번째数表示有多少번째并열判断，元素的배열内的元素为并열判断 |
| logic | array | while判断与while判断之间的关系，按배열内元素顺序；1：或，0：与 |
| logicGroup | array | conditionGroup节点的元素的判断条件；1：或，0：与，null表示该层没有判断 |

**예시**：

```json
{
    "logout": false,
    "type": 13,
    "userParamInt": 0,
    "userParamString": "",
    "whileconditionArrayFINAL": {
        "conditionGroup": [
            [
                {
                    "logicType": 1,
                    "paraGroupOne": {
                        "data": 0.0,
                        "secondvalue": 0,
                        "value": 1,
                        "varname": "I002"
```

## 等到모션到点 (WAIT_POS)

**설명**：MOV명령어실행종료并不代表서보실행종료，仅为포인트下发종료，此명령어대기서보모터실행精准到点，再실행下一번째명령어。此명령어필수在모션명령어后실행此명령어才生效。此명령어只能仅支持前台프로그램，后台없음모션명령어自然也不需要使用。此명령어不支持提前실행。如果로봇在最小대기时间和最大대기时间之间，모션到点了，就会종료대기，실행下一条명령어。

**명령어名**：WAIT_POS

| 매개변수名 | 유형 | 필수 | 설명 |
|--------|------|------|------|
| logout | bool | 아니오 | 주석 처리 여부；true：在작업파일中将不실행该명령어；false：未주석 처리됨，실행 가능 |
| type | int | 예 | WAIT_POS在열거형数열中为238 |
| userParamInt | int | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 아니오 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| accurary | double | 예 | 精도；[0.0001,0.1]도/s |
| waitType | int | 아니오 | 대기유형；0:위치，1：속도 |
| maxTime | int | 예 | 最大대기时间；[20,20000] |
| minTime | int | 예 | 最小대기时间；[20,20000] |

**예시**：

```json
{
    "logout": false,
    "type": 238,
    "userParamInt": 0,
    "userParamString": "",
```
