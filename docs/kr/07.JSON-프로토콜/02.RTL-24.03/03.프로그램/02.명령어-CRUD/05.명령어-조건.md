# 명령어-条件제어类

## 호출子프로그램 (CALL)

**설명**：호출另一个프로그램，被호출的프로그램실행完后반환主프로그램CALL명령어的下一行继续실행。

**명령어名**：CALL

| 매개변수名 | 유형 | 必填 | 설명 |
|--------|------|------|------|
| callInParaVarData | array | 否 | 输入매개변수个数及选择；有0-12个元素。元素格式：`{"data":0.0,"secondvalue":0,"value":2,"varname":"D001"}` |
| callOutParaVarData | array | 否 | 输出매개변수个数及选择；有0-12个元素。元素格式：`{"data":0.0,"secondvalue":0,"value":2,"varname":"D001"}` |
| jobName | string | 是 | 호출프로그램名称，不能是本身，호출的프로그램中须有매개변수声明 |
| logout | bool | 否 | 주석 처리 여부；true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 |
| type | int | 是 | CALL在枚举数列中为15 |
| userParamInt | int | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |

**예시**：有两个매개변수

```json
{
    "callInParaVarData": [
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 2,
            "varname": "D001"
        },
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 2,
            "varname": "D002"
        }
    ],
    "callOutParaVarData": [
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 5,
            "varname": "GD003"
        },
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 5,
            "varname": "GD004"
        }
    ],
    "jobName": "QQQQ",
    "logout": false,
    "type": 15,
    "userParamInt": 0,
    "userParamString": ""
}
```

## 退出 (RETURN)

**설명**：退出当前실행的작업파일，例如：子프로그램、后台프로그램。

**명령어名**：RETURN

| 매개변수名 | 유형 | 必填 | 설명 |
|--------|------|------|------|
| logout | bool | 否 | 주석 처리 여부；true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 |
| type | int | 是 | RETURN在枚举数列中为251 |
| userParamInt | int | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |

**예시**：

```json
{
    "logout": false,
    "type": 251,
    "userParamInt": 0,
    "userParamString": ""
}
```

## 호출Lua파일 (CALL_LUAFILE)

**설명**：매개변수定义好之后，直接上传Lua파일，然后插入호출Lua파일명령어，명령어실행结束后会把写好的所有매개변수변수传入和输出。

**명령어名**：CALL_LUAFILE

| 매개변수名 | 유형 | 必填 | 설명 |
|--------|------|------|------|
| logout | bool | 否 | 주석 처리 여부；true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 |
| type | int | 是 | CALL_LUAFILE在枚举数列中为218 |
| userParamInt | int | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| inParaVarData | array | 否 | 传入매개변수个数及选择；元素个数[0-12]。元素格式：`{"data":0.0,"secondvalue":0,"value":1,"varname":"I001"}` |
| luafileName | string | 是 | 호출的lua파일 |
| outParaVarData | array | 否 | 输出매개변수个数及选择；元素个数[0-12]。元素格式：`{"data":0.0,"secondvalue":0,"value":1,"varname":"I001"}` |
| textEditType | bool | 否 | 文本编译유형；true：文本유형，false：语句유형 |

**예시**：

```json
{
    "logout": false,
    "type": 218,
    "userParamInt": 0,
    "userParamString": "",
    "inParaVarData": [
        {
            "data": 0.0,
            "secondvalue": 0,
            "value": 1,
            "varname": "I001"
        },
        {
            "data": 2.0,
            "secondvalue": 0,
            "value": 0,
            "varname": ""
        }
    ],
    "luafileName": "demo.lua",
    "outParaVarData": [
        {
            "data": 3.0,
            "secondvalue": 0,
            "value": 0,
            "varname": ""
        },
        {
            "data": 6.0,
            "secondvalue": 0,
            "value": 0,
            "varname": ""
        }
    ],
    "textEditType": true
}
```

## 如果 (IF)

**설명**：通过判断条件是否成立来执行下一目标的명령어，判断条件不满足时，执行ELSEIF语句。IF和ENDIF配对使用，不能单独删除或更改，会导致프로그램실행出错。ELSE IF不可脱离IF单独使用。ELSE不可脱离IF单独使用。

**명령어名**：IF

| 매개변수名 | 유형 | 必填 | 설명 |
|--------|------|------|------|
| logout | bool | 否 | 주석 처리 여부；true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 |
| type | int | 是 | IF在枚举数列中为9 |
| userParamInt | int | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| ifconditionArrayFINAL | object | 是 | 表示if判断；{}内的为if判断的데이터 |

**ifconditionArrayFINAL 结构**：

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| conditionGroup | array | 元素个数表示有多少个判断 |
| logic | array | if判断与if判断之间的关系，按数组内元素顺序；1：或，0：与 |
| logicGroup | array | conditionGroup节点的元素的判断条件；1：或，0：与，null表示该层没有判断 |

**conditionGroup 元素结构**：

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| logicType | int | if判断条件比较方式；1：==，2：<，3：>，4：<=，5：>=，6：!= |
| paraGroupOne | object | 변수1，格式：`{"data":0.0,"secondvalue":0,"value":1,"varname":"I002"}` |
| paraGroupTwo | object | 변수2，格式：`{"data":0.0,"secondvalue":0,"value":1,"varname":"I002"}` |

**예시**：

## 等待 (WAIT)

**설명**：条件成立之前프로그램处于等待상태，条件成立后继续执行等待명령어后面的명령어。

**명령어名**：WAIT

| 매개변수名 | 유형 | 必填 | 설명 |
|--------|------|------|------|
| logout | bool | 否 | 주석 처리 여부；true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 |
| type | int | 是 | WAIT在枚举数列中为8 |
| userParamInt | int | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |

| paraTime | object | 否 | 等待时间；double유형；[0.01,9999]s |
| para_wait_result_ | object | 否 | 等待结果，满足条件为true；bool유형 |
| waitConditionFinal | object | 否 | {}内为参与判断的条件和변수 |
| wait_value_check_ | bool | 否 | 是否接受结果；true：存在"para_wait_result_"节点，false：不存在 |

**waitConditionFinal 结构**：

## WHILE 循环 (WHILE)

**설명**：判断条件为真是反复执行循环语句里面的명령어。插入WHILE명령어的同时会同时插入ENDWHILE명령어。若要删除WHILE명령어请同时删掉其对应的ENDWHILE명령어，否则会导致프로그램无法실행。当프로그램的开头为WHILE且最后一样명령어为ENDWHILE时，请在프로그램的开头或结尾插入一条TIMER（延时）명령어。否则当WHILE명령어的条件不满足时会导致프로그램陷入死机。当WHILE内部的명령어没有모션类명령어或在某种情况下可能会陷入死循环时，请在WHILE与ENDWHILE间插入一条TIMER（延时）명령어，否则当WHILE명령어的条件满足时可能会导致프로그램陷入死机。当WHILE명령어的条件满足时，会循环실행WHILE与ENDWHILE两条명령어之间的명령어。在실행到WHILE명령어之前若判断条件不满足，在실행到WHILE명령어时会直接跳转到ENDWHILE명령어而不실행WHILE与ENDWHILE之间的명령어；若在실행WHILE与ENDWHILE之间的명령어过程中，判断条件变成不满足，会继续실행，直到실행到ENDWHILE行，不再循环而是继续실행ENDWHILE下面的명령어。

**명령어名**：WHILE

| 매개변수名 | 유형 | 必填 | 설명 |
|--------|------|------|------|
| logout | bool | 否 | 주석 처리 여부；true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 |
| type | int | 是 | WHILE在枚举数列中为13 |
| userParamInt | int | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| whileconditionArrayFINAL | object | 是 | 表示while判断；{}内的为while判断的데이터 |

**whileconditionArrayFINAL 结构**：

| 매개변수名 | 유형 | 설명 |
|--------|------|------|
| conditionGroup | array | 元素个数表示有多少个并列判断，元素的数组内的元素为并列判断 |
| logic | array | while判断与while判断之间的关系，按数组内元素顺序；1：或，0：与 |
| logicGroup | array | conditionGroup节点的元素的判断条件；1：或，0：与，null表示该层没有判断 |

## 等到모션到点 (WAIT_POS)

**설명**：MOV명령어执行结束并不代表伺服실행结束，仅为点位下发结束，此명령어等待伺服电机실행精准到点，再执行下一个명령어。此명령어必须在모션명령어后执行此명령어才生效。此명령어只能仅支持前台프로그램，后台无모션명령어自然也不需要使用。此명령어不支持提前执行。如果로봇在最小等待时间和最大等待时间之间，모션到点了，就会退出等待，执行下一条명령어。

**명령어名**：WAIT_POS

| 매개변수名 | 유형 | 必填 | 설명 |
|--------|------|------|------|
| logout | bool | 否 | 주석 처리 여부；true：在작업파일中将不执行该명령어；false：未주석 처리됨，可执行 |
| type | int | 是 | WAIT_POS在枚举数列中为238 |
| userParamInt | int | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| userParamString | string | 否 | 2차 개발에서 사용자 정의, 현재 의미 없음 |
| accurary | double | 是 | 精度；[0.0001,0.1]度/秒 |
| waitType | int | 否 | 等待유형；0:위치，1：속도 |
| maxTime | int | 是 | 最大等待时间；[20,20000] |
| minTime | int | 是 | 最小等待时间；[20,20000] |

**예시**：

```json
{
    "logout": false,
    "type": 238,
    "userParamInt": 0,
    "userParamString": "",
    "accurary": 0.10,
    "maxTime": 200,
    "minTime": 20,
    "waitType": 0
}
```

## 충돌 감지 (DETECTCOLLISION_SET / DETECTCOLLISION_RESET)

**설명**：충돌 감지설정：호출临时매개변수，方便及时调整충돌 감지阈值。충돌 감지설정명령어只是临时매개변수，只对충돌 감지설정명령어下面的모션명령어有效，不插入충돌 감지설정명령어时호출人机协作-力学기능界面的충돌 감지阈值（명령어）매개변수。충돌 감지매개변수复位：复位临时使用的충돌 감지阈值，执行此条명령어后会호출人机协作-力学기능界面的충돌 감지阈值（명령어）매개변수。
