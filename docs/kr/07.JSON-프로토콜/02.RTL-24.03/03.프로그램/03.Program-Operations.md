# 프로그램操作

## 프로그램열기

正式실행프로그램之前需要열기프로그램。

**명령어:** `0x5100`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要操作的로봇 번호，取值범위 [1, 4] |
| jobName | string | 新建작업 파일名 |
| suffixname | string | 文件확장자：.JBR 前台프로그램，.JBP 后台局部프로그램，.JBPG 后台全局프로그램 |

```json
{
  "robot": 1,
  "jobName": "xxxx",
  "suffixname": ".JBR"
}
```

## 프로그램복사

복사后프로그램的后缀예一样的。

**명령어:** `0x5101`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要操作的로봇 번호，取值범위 [1, 4] |
| oldjobName | string | 旧파일명 |
| newjobName | string | 복사后파일명 |
| suffixname | string | 文件확장자：.JBR 前台프로그램，.JBP 后台局部프로그램，.JBPG 后台全局프로그램 |

```json
{
  "robot": 1,
  "oldjobName": "QQQQ",
  "newjobName": "WWWW",
  "suffixname": ".JBR"
}
```

## 프로그램해제

会해제현재로봇的所有프로그램，但不会해제리셋프로그램。

**명령어:** `0x5102`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要조회的로봇 번호，取值범위 [1, 4] |
| lock_enable | bool | 예아니오清空작업 파일활성화，true 清空활성화，false 清空不활성화 |

```json
{
  "robot": 1,
  "lock_enable": true
}
```

**명령어:** `0x5103`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要操作的로봇 번호，取值범위 [1, 4] |

```json
{
  "robot": 1
}
```

**컨트롤러 응답**

**명령어:** `0x5104`

| 필드 | 유형 | 설명 |
|------|------|------|
| openedJobName | string | 현재열기的작업 파일，空表示没有 |

```json
{
  "openedJobName": "asd"
}
```

## 主프로그램

主프로그램可以认为예特殊的前台프로그램，只有前台프로그램可以설정成主프로그램。

### 설정작업 파일为主프로그램

**명령어:** `0x5110`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要조회的로봇 번호，取值범위 [1, 4] |
| jobName | string | 작업 파일名，空문자열表示取消主프로그램 |

```json
{
  "robot": 1,
  "jobName": "xxx"
}
```

### 조회主프로그램

**명령어:** `0x5111`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要조회的로봇 번호，取值범위 [1, 4] |

```json
{
  "robot": 1
}
```

**컨트롤러 응답**

**명령어:** `0x5112`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要조회的로봇 번호，取值범위 [1, 4] |
| jobName | string | 작업 파일名，空表示没有 |

```json
{
  "robot": 1,
  "jobName": "xxx"
}
```

## 프로그램저장

프로그램进줄수정、삽입、삭제后需要使用该协议进줄저장。

**명령어:** `0x5120`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要조회的로봇 번호，取值범위 [1, 4] |
| jobname | string | 작업 파일名 |
| suffix | string | 文件확장자：.JBR 前台프로그램，.JBP 后台局部프로그램，.JBPG 后台全局프로그램 |
| time | string | 타임스탬프，형식如 "2024.08.08 14:16:39" |

```json
{
  "robot": 1,
  "jobname": "xxxx",
  "suffix": ".JBR",
  "time": "2024.08.08 14:16:39"
}
```

## 同步프로그램열기

> 功能可能有问题，只有在 6001 열기时同步到 6000 的操作

컨트롤러열기프로그램后通知호스트 컴퓨터，告知호스트 컴퓨터该프로그램已在컨트롤러열기。

**명령어:** `0x3130`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 현재反馈的로봇 번호，取值범위 [1, 4] |
| jobName | string | 작업 파일名 |

```json
{
  "robot": 1,
  "jobName": ""
}
```

## 전송프로그램명령어

为없음작업 파일的명령어삽입。실행方式与작업실행方式一样，但예需要将 "noFileRun" 节点的值写为 true。

**명령어:** `0x5122`

| 필드 | 유형 | 설명 |
|------|------|------|
| robot | int | 要操作的로봇 번호，取值범위 [1, 4] |
| cmd | object | 삽입的명령어详情，형식为 cmd->toJson()，见명령어页面 |

```json
{
  "robot": 1,
  "cmd": {}
}
```

## 프로그램同步

호스트 컴퓨터 전송给컨트롤러，通知티치 펜던트同步작업 파일。

**명령어:** `0x1222`

- 없음需전송数据

**컨트롤러响应**

**명령어:** `0x1223`

| 필드 | 유형 | 설명 |
|------|------|------|
| finished | bool | true 성공，false 실패 |

```json
{
  "finished": true
}
```

### 컨트롤러通知티치 펜던트同步작업 파일

**명령어:** `0x1224`

- 없음需전송数据

**티치 펜던트响应**

**명령어:** `0x1225`

| 필드 | 유형 | 설명 |
|------|------|------|
| finished | bool | true 성공，false 실패 |

```json
{
  "finished": true
}
```
