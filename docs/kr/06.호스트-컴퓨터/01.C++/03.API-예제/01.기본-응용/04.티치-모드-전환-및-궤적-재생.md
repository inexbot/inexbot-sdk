# 4. 示教 모드 유형 전환 및 轨跡 재생 기능

상위컴퓨터(上位機)를 사용하여 拖拽 모드를 활성화하기 전에 먼저 示教盒(티치 펜던트)에서 拖拽 식별(identification)을 수행해야 합니다.

주의: 拖拽 모드로 전환한 후 set_servo_poweron 인터페이스를 호출하여 서보 上使能을 활성화해야 拖拽을 진행할 수 있습니다.

## 1. 示教 모드 유형 전환 함수

```cpp
/*
 * 示教 모드 유형 전환 함수 캡슐화
 */
void Switching_teach_type(int fd)
{
  int type;
  get_teach_type(fd, type);
  switch (type)
  {
  case 0:   //현재 示教 모드 유형이 点动 모드
    set_teach_type(fd, 1);   //현재 示教 모드 유형을 拖拽 모드로 전환
    break;
  
  case 1:   //현재 示教 모드 유형이 拖拽 모드
    set_teach_type(fd, 0);   //현재 示教 모드 유형을 点动 모드로 전환
    break;
  }


}
```

## 2. 모션 종료 대기 함수 캡슐화

```cpp
/*
 * 모션 종료 시까지 루프 블로킹 함수
 */
void wait_for_running_over(int fd)
{
    // 모션 완료 대기
    int running_state = 0;
    get_robot_running_state(fd, running_state);  // 로봇이 모션 중인지 조회 2-모션 중
    while (running_state == 2)
    {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));  // 500ms 블로킹
        get_robot_running_state(fd, running_state);   // 다시 조회
    }
}
```

## 4. 拖拽 轨 기록 함수 캡슐화

```cpp
/*
 * 拖拽 轨 기록 함수 캡슐화
 */
void track_record(int fd)
{
  set_teach_type(fd, 1);   //현재 示教 모드 유형을 拖拽 모드로 전환
  double maxSamplingNum = 200;     //轨跡의 샘플링 간격
  double samplingInterval = 0.05;  //최대 샘플링 포인트 수
  int vel = 30;                    //轨跡 재생 속도
  track_record_delete(fd);         //새 轨跡 기록 전 기존 轨跡 제거
  track_record_start(fd, maxSamplingNum, samplingInterval);         //轨跡 기록 시작
  for(int i = 0; i < 50; i++)
  {
    sleep(0.2);     ////.....10초 대기, 拖拽 종료 대기
  }
  track_record_stop(fd);                                            //轨跡 기록 종료
}
```
