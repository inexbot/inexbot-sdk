# API 页面模板

> 本文档是 SDK API 文档页面的写作规范和模板。编写和审核 API 页面时请参照此模板。

---

## 模板结构

每个 API 页面应包含以下章节（按顺序）：

```text
# 功能名称
## 概述
## 版本信息
## 函数签名
## 参数说明
## 返回值
## 错误码
## 前置条件
## 线程安全与实时性
## 示例
## 注意事项
## 相关接口
```

---

## 各章节编写要求

### 概述

- 用 1-2 句话说明该 API 的功能和应用场景
- 如果是运动相关接口，标注 **是否会导致机器人实际运动**（⚠️ 或 ✅ 标识）

### 版本信息

| 项目 | 说明 |
|------|------|
| 适用 RTL 版本 | RTL-22.07 / RTL-24.03 |
| SDK 最低版本 | 如 v1.2.0+ |
| 可用语言 | C++ / C# / Python（标注各语言是否均支持） |

### 函数签名

按语言分别列出函数签名：

**C++**

```cpp
int connect_robot(const char* ip, const char* port);
```

**Python**

```python
def connect_robot(ip: str, port: str) -> int:
```

### 参数说明

| 参数名 | 类型 | 输入/输出 | 取值范围 | 说明 |
|--------|------|----------|---------|------|
| ip | const char* | 输入 | 有效的 IPv4 地址 | 控制器 IP 地址 |
| port | const char* | 输入 | "6001" | 控制器端口号 |

### 返回值

| 返回值 | 含义 |
|--------|------|
| > 0 | 连接成功，返回 socket 文件描述符 |
| <= 0 | 连接失败 |

### 错误码

| 错误码 | 含义 | 可能原因 |
|--------|------|---------|
| -1 | 连接超时 | 网络不通或 IP 错误 |
| -2 | 端口不可用 | 端口被占用或防火墙拦截 |

### 前置条件

- 控制器已上电且网络可达
- （运动接口）伺服已使能，急停已释放
- （运动接口）工作空间内无人员或障碍物

### 线程安全与实时性

- **线程安全：** 标注是否可在多线程环境中调用
- **阻塞行为：** 标注是否会阻塞调用线程及阻塞时间
- **实时性：** 标注是否可在实时任务中调用

### 示例

完整可运行的代码示例（包含必要的 `#include` 或 `import`）：

**C++ 示例**

```cpp
#include "cpp_interface/nrc_api.h"
#include <iostream>

int main() {
    SOCKETFD fd = connect_robot("192.168.1.15", "6001");
    if (fd <= 0) {
        std::cout << "连接失败" << std::endl;
        return 1;
    }
    std::cout << "连接成功" << std::endl;
    return 0;
}
```

**预期输出：**

```text
连接成功
```

### 注意事项

- 运动接口必须标注安全警告 ⚠️
- 标注坐标系类型（关节坐标 / 笛卡尔坐标 / 工具坐标）
- 标注单位（mm、度、百分比 等）
- 标注已知限制和兼容性问题

### 相关接口

- 列出功能相关的前置或后继 API 调用链接

---

## 运动接口补充要求

所有会导致机器人实际运动的接口，除上述章节外，还需增加：

### 坐标系与单位

| 项目 | 说明 |
|------|------|
| 坐标系 | 关节坐标（coord=0）/ 笛卡尔坐标（coord=1）/ 工具坐标（coord=2） |
| 位置单位 | 关节角为度（°），笛卡尔位置为毫米（mm） |
| 速度单位 | 百分比（%）或 毫米/秒（mm/s） |

### 安全条件 ⚠️

> **安全警告：** 调用此接口会导致机器人实际运动！
> - 确保工作空间内无人员和障碍物
> - 确认急停按钮在可触及范围内
> - 首次运行时使用低速参数
