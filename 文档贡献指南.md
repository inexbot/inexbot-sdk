# 文档贡献指南

> 本文档定义纳博特 SDK 文档的中文术语、写作格式和文件组织规范。
> 所有贡献者应遵循本指南，以保持多语言文档的一致性。

---

## 2. 中英文混排规范

### 2.1 中文与英文之间加空格

**规则：** 中文与英文单词之间、中文与数字之间应加一个半角空格。

```markdown
✅ 正确：这是一个 Python SDK 的快速开始指南。
✅ 正确：支持 Windows 10 及以上版本。
✅ 正确：请使用 TCP 端口 6000 进行连接。

❌ 错误：这是一个Python SDK的快速开始指南。
❌ 错误：支持Windows10及以上版本。
❌ 错误：请使用TCP端口6000进行连接。
```

### 2.2 例外情况

以下情况**不加空格**：

- 中文标点与英文之间：`使用Python、C++和C#。`
- 英文单词内部的连字符/斜线：`EtherCAT-based`、`C++`
- 括号内的全英文内容：`请参考上节（Getting Started）`
- 版本号：`v2.0.0`、`RTL-24.03`
- 文件路径：`docs/zh/index.md`
- 代码或行内代码：`` `npm run docs:build` ``

### 2.3 标点符号

- 中文文档使用中文标点（，。！？、：“”『』（））
- 代码段落、命令行示例内部使用英文标点
- 英文术语与中文之间使用中文标点

```markdown
✅ 支持平台：Windows、Linux 和 macOS。
✅ SDK 支持 C++、C# 和 Python 三种语言。
```

## 3. 标题层级规范

### 3.1 一级标题

- **每页有且只有一个一级标题**（`#`），即页面标题。
- 一级标题应准确概括页面内容。

```markdown
# Python 快速开始 ✅ 正确

# 快速开始 ✅ 正确

# 安装依赖 ❌ 不应作为 H1（应作为 H2）

# 介绍 ❌ 不应作为 H1

# 概述 ❌ 不应作为 H1

## 介绍 ✅ H2 级别
```

### 3.2 标题层级递进

- 标题层级只能逐级递增，不能跳级（H1 → H2 → H3，不能 H1 → H3）。
- 不同章节的标题在同一层级。

```markdown
# 页面标题

## 环境要求 ✅ H2

### 操作系统 ✅ H3

#### Windows ✅ H4

#### Linux ✅ H4

## 安装步骤 ✅ H2

### 下载 SDK ✅ H3
```

### 3.3 标题格式

- 标题前后应有空行（markdownlint 规则 MD022）。
- 标题内容与 `#` 之间加一个空格。
- 标题不用句号结尾。

## 4. 代码块规范

### 4.1 必须指定语言标识
每个围栏代码块（` ``` `）必须指定语言标识，不允许裸 ` ``` `。

✅ 正确写法—指定语言标识：

```python
import nrc
```

```json
{
    "method": "moveJ"
}
```

❌ 错误写法—缺少语言标识：

```
import nrc
```

### 4.2 常用语言标识

| 内容 | 标识 | 说明 |
|------|------|------|
| Python | `python` | Python 代码 |
| C++ | `cpp` | C++ 代码 |
| C# | `csharp` | C# 代码 |
| JSON | `json` | JSON 数据 |
| Shell | `bash` | 命令行 |
| 协议示例 | `text` | 纯文本协议示例 |
| 配置文件 | `yaml` | YAML/yml 配置 |
| 非代码内容 | `text` | 非代码的输出示例 |

### 4.3 代码块格式

- 代码块前后应有空行。
- 代码内容不应超出 80 列（建议）。
- 命令行示例不加 `$` 前缀（避免误导复制），如需区分输入输出可使用注释。

## 5. 警告与提示格式

### 5.1 使用 VitePress 容器

优先使用 VitePress 内置的 `:::` 容器语法，而非 `> blockquote`：

```markdown
::: warning
这里写警告内容。
:::

::: tip
这里写提示内容。
:::

::: info
这里写说明内容。
:::

::: danger
这里写危险警告，仅用于可能导致设备损坏或人身安全的情况。
:::
```

### 5.2 中文容器标题

```markdown
::: warning 注意
连接前请确认控制器处于急停状态。
:::

::: tip 提示
首次使用建议先运行只读示例。
:::

::: danger 安全警告
运动示例可能导致设备损坏或人身伤害。
:::
```

### 5.3 使用场景

| 容器          | 用途                   | 示例场景                     |
| ------------- | ---------------------- | ---------------------------- |
| `::: warning` | 潜在风险，需要用户注意 | 端口说明、版本兼容、配置注意 |
| `::: tip`     | 有用提示，非必须但推荐 | 快捷操作、最佳实践           |
| `::: info`    | 补充说明               | 参数说明、行为解释           |
| `::: danger`  | 严重安全警告           | 运动安全、高压、急停         |

### 5.4 兼容旧格式

对于大规模的 `> 说明：` 和 `> 注意：` 格式，在重构时可逐步迁移。新文档直接使用 `:::` 容器。

## 6. 文件命名与链接规范

### 6.1 文件命名规则

```
{序号}.{名称}.md
```

- **序号**：两位数字，从 `00` 开始，不重复
- **名称**：中文文档使用中文名称，英文文档使用英文名称
- **分隔符**：目录名和文件名中单词之间使用连字符 `-`
- **index.md**：每个目录必须有 `index.md` 作为入口页

```markdown
✅ 正确：01.快速开始.md 02.入门指南.md 06.上位机/01.C++/01.渐入式教程/初始化项目.md

❌ 错误：
1.quickstart.md
C++初始化项目.md（缺少序号前缀）
```

### 6.2 目录结构规范

```
docs/zh/
├── index.md                          # 首页
├── 01.概述.md                        # 顶层页面
├── 02.入门指南.md
├── 04.控制器/                        # 目录
│   ├── index.md                      # 控制器首页
│   ├── 01.快速开始.md                # 子页面
│   └── 02.二次开发API.md
└── 06.上位机/
    ├── index.md
    ├── 01.C++/
    │   ├── index.md
    │   └── 01.渐入式教程/
    │       ├── index.md
    │       └── 初始化项目.md
```

### 6.3 链接规范

- **站内链接**：使用相对路径，不加 `.md` 后缀或 `.html` 后缀
- **目录页链接**：链接到目录名即可（自动解析到 index.md）
- **跨语言链接**：不跨语言链接，各语言版本独立维护

```markdown
✅ 正确：`[快速开始](./04.控制器/01.快速开始)`
✅ 正确：`[Python 指南](06.上位机/03.Python/01.渐入式教程)`

❌ 错误：`[快速开始](./04.控制器/01.快速开始.md)`
❌ 错误：`[快速开始](./04.控制器/01.快速开始.html)`
❌ 错误：`[快速开始](./04.控制器/01.快速开始#安装)`
```

### 6.4 图片资源规范

- 图片放入当前目录的 `assets/` 子目录
- 图片文件名使用英文小写 + 连字符：`quick-start-01.png`
- 必须填写 alt 文本，示例：`` ![控制器连接示意图](quick-start/connect.png) ``（反引号包裹）
- 支持格式：PNG（截图）、SVG（图表）、WebP（照片）

## 7. 多语言同步规范

> 中文为权威源，英文和韩文基于中文同步。

1. **中文先行**：先完成中文文档的编写和审核
2. **结构对齐**：三语言的文件结构应保持一致（序号、目录层级）
3. **术语映射**：中文术语 ↔ 英文术语 ↔ 韩文术语应建立对照表
4. **同步记录**：每页记录最后同步日期，便于跟踪差异

## 8. 检查清单

提交文档修改前，请逐项确认：

- [ ] 术语大小写符合规范（API / SDK / EtherCAT / JSON 等）
- [ ] 中文与英文之间有空格
- [ ] 页面有且只有一个一级标题
- [ ] 标题层级递进正确（不跳级）
- [ ] 代码块指定了语言标识
- [ ] 警告使用 `:::` 容器而非 `>` 引用
- [ ] 站内链接使用相对路径，不带 `.md` 后缀
- [ ] 图片有 alt 文本且文件存在
- [ ] `npm run docs:check` 通过
- [ ] `npm run docs:build` 通过
