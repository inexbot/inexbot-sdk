# 文档贡献指南

本文档定义了纳博特 SDK 文档的写作规范和格式标准。所有文档贡献者在提交前请仔细阅读并遵循。

---

## 1. 术语与大小写规范

### 1.1 核心技术名词

下表列出文档中的核心术语及其正确写法：

| 术语 | 正确写法 | 错误写法（禁止使用） |
|------|---------|---------------------|
| API | API | api、Api |
| SDK | SDK | sdk、Sdk |
| Demo | Demo | demo、DEMO |
| Python | Python | python、PYTHON |
| C++ | C++ | c++、Cpp、cpp |
| C# | C# | c#、Csharp、csharp |
| JSON | JSON | Json、json |
| EtherCAT | EtherCAT | Ethercat、ethercat、EtherCat |
| MoveIt 2 | MoveIt 2 | moveit2、MoveIt2、moveit |
| ROS | ROS | Ros、ros |
| ROS 2 | ROS 2 | ros2、ROS2 |
| RTL | RTL | Rtl、rtl |
| Qt | Qt | QT、qt |
| MinGW | MinGW | mingw、MinGW64（系统名称除外） |
| MSVC | MSVC | msvc |

### 1.2 其他规范

> **注意：** 术语表中"错误写法"列的内容（如 `json`、`sdk`、`api` 等）是刻意展示的错误示例，用于教学目的，非实际文档错误。

- **Linux**、**Windows**：作为操作系统名称时首字母大写
- **TCP**、**HTTP**：协议名称全大写
- **x64**、**x86**、**ARM64**：架构标识小写
- **.NET**：始终以点开头，大写 NET

## 2. 中英文排版规范

### 2.1 中英文之间空格

中文与英文单词、数字之间应添加半角空格：

- ✅ `Python SDK 支持 Windows 和 Linux 平台`
- ❌ `PythonSDK支持Windows和Linux平台`

### 2.2 中文与数字之间空格

- ✅ `6000 端口`、`7 轴`
- ❌ `6000端口`、`7轴`

### 2.3 中英文符号

- 中文句子中使用中文标点（全角），如：、。！？
- 英文和代码中使用英文标点（半角），如：, . ! ?
- 中文句子中引用代码或技术名词时保留原样：

  ✅ `使用 `connect_robot` 函数建立连接`
  ❌ `使用“connect_robot”函数建立连接`

## 3. 标题层级规范

### 3.1 页面内标题

- 页面只有一个一级标题（`#`），对应页面主题
- 使用 `##`、`###`、`####` 逐级嵌套，不要跳级
- 标题中不应包含链接或代码块
- 标题末尾不加标点符号

### 3.2 目录页标题

目录页（`index.md`）的一级标题应简明概括本章内容：

- ✅ `# Python`、`# JSON 协议`
- ❌ `# Python SDK 开发文档详细说明`

## 4. 代码块规范

### 4.1 语言标识

所有代码块必须标注语言标识：

```cpp
// C++ 代码
int main() { return 0; }
```

```python
# Python 代码
print("Hello")
```

```text
纯文本内容（目录结构、输出示例等）
```

### 4.2 目录结构用 text

目录树结构使用 `text` 而非代码语言：

✅ 
```text
项目/
├── src/
│   └── main.cpp
└── libs/
```

❌ 
```cpp
项目/
├── src/
└── libs/
```

## 5. 提示与警告格式

推荐使用 VitePress 自定义容器代替旧式 Markdown 块引用：

| 用途 | 推荐写法 | 旧写法（不推荐） |
|------|---------|----------------|
| 提示/说明 | `::: tip` 或 `::: info` | `> **说明**：` |
| 警告 | `::: warning` | `> **注意**：` |
| 危险 | `::: danger` | `> **安全警告**：` |
| 版本信息 | `::: info 版本` | `> **版本说明**：` |

示例：

```markdown
::: tip 提示
SDK 版本需与控制器固件版本匹配。
:::

::: warning 注意
首次连接前请确认网络可达。
:::

::: danger ⚠️ 安全警告
以下代码会导致机器人实际运动！
:::
```

> 已有文档中的旧式 `> **注意**：` 格式可逐渐迁移，新页面统一使用 `:::` 容器。

## 6. 文件命名与链接规范

### 6.1 文件命名

- 使用中文文件名，简洁明了
- 目录索引文件统一命名为 `index.md`
- 文件名避免使用空格和特殊字符
- 示例：`01.渐入式教程`、`02.接口示例`

### 6.2 内部链接

- 使用相对路径：`./index.md`、`../02.入门指南.md`
- 链接到目录时指向 `index.md`
- 不要在链接中使用绝对路径
- 包含括号的路径需做 URL 编码：`%28` 代替 `(`，`%29` 代替 `)`

### 6.3 外部链接

使用完整的 URL，确保链接可访问。

## 7. 版本标记规范

在页面中标记适用版本时，使用统一格式：

```markdown
> **适用版本：** RTL-22.07 / RTL-24.03
```

或在章节开头说明：

```markdown
## 功能名称（RTL-24.03 新增）
```

---

## 8. 图片规范

- 中文文档图片存放在 `docs/zh/assets/` 目录
- 使用相对路径引用：`![描述](assets/image.png)`
- 图片文件名使用英文小写 + 连字符

---

## 9. 运动接口补充要求

所有会导致机器人实际运动的接口页面，必须包含：

1. **醒目安全警告**（使用 `::: danger` 容器）
2. **坐标系与单位说明**
3. **前置条件**（伺服使能、急停释放等）
4. **建议参数**（低速、安全位置）

详见 [API 页面模板](./API页面模板.md)。
